# Philosophy

> Engineering Project Template is built around one idea: project documentation should remain simple, modular and easy to navigate for both humans and AI agents.

---

# Mission

Engineering Project Template provides a lightweight documentation standard that enables projects to start with only what they need and evolve progressively as complexity grows.

The objective is not to maximize documentation.

The objective is to maximize clarity.

---

# Core Principles

## Simple by Default

Projects should begin with the smallest useful documentation set.

Additional documents are introduced only when they provide clear value.

---

## Progressive Growth

Documentation evolves together with the project.

The template must never encourage creating documents "just in case."

---

## Single Source of Truth

Every piece of information has exactly one authoritative location.

Duplication is avoided.

Documents reference each other instead of repeating content.

---

## Documentation as Navigation

Documentation is a navigation system rather than a collection of isolated files.

Every document helps readers identify the next relevant document.

---

## Human and AI Collaboration

The documentation is designed to be equally understandable by humans and AI agents.

Both should reach the required information with minimal cognitive effort.

---

## Tool Independence

Projects must remain independent from specific AI assistants or technologies.

Tool-specific integration documents are adapters, never authoritative sources.

---

## Long-Term Maintainability

The standard should evolve cautiously.

New documents are introduced only after repeated validation across multiple independent projects.

---

# What Engineering Project Template Is Not

Engineering Project Template is not:

* a software framework;
* a project generator;
* a development methodology;
* a coding standard;
* a replacement for the Engineering Playbook.

---

# Relationship with the Engineering Playbook

Engineering Playbook defines how projects are developed.

Engineering Project Template defines how project knowledge is organized.

Together they create a consistent engineering ecosystem while remaining independently versioned and governed.
