# Decisions

> Preserve important project decisions and their rationale.

---

## Purpose

Record significant project decisions together with the reasons behind them.

This document explains **why** a decision was made.

It does not describe implementation details.

---

## Use when

* Making an important architectural decision.
* Choosing between multiple approaches.
* Understanding why something exists.
* Reviewing previous project choices.

---

## Related

* `PRODUCT_VISION.md`

# Decision Log

Each decision should use the following structure.

---

## DEC-001 — Decision Title

| Property          | Value                                       |
| ----------------- | ------------------------------------------- |
| Status            | Proposed / Accepted / Superseded / Rejected |
| Date              |                                             |
| Author            |                                             |
| Related Documents |                                             |

### Context

Describe the problem or situation that required a decision.

### Decision

Describe the chosen solution.

### Rationale

Explain why this solution was selected.

### Consequences

Describe the expected positive and negative impacts.

### Alternatives Considered

List the main alternatives and briefly explain why they were not selected.

---

## DEC-002

Repeat the same structure for each new decision.
